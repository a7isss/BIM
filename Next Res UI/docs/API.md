# Next Res Engine API Documentation

This document describes the API endpoints provided by the development server (currently implemented as Vite middleware in `vite.config.ts`).

> **Note:** These endpoints are currently only available during development via the Vite dev server. In Sprint 6, they will be extracted to a standalone backend server.

## Endpoints

### 1. `GET /api/project`
Loads the current project data.
- **Response**: `200 OK` with JSON payload matching the `ResPlanData` schema (nodes, elements, slabs, etc.).
- **Error**: `500 Internal Server Error` if the file cannot be read.

### 2. `POST /api/project`
Saves the current project data.
- **Body**: JSON payload matching the `ResPlanData` schema.
- **Response**: `200 OK` with `{"success": true}`.
- **Error**: `500 Internal Server Error` if the file cannot be written.

### 3. `GET /api/types`
Loads the architectural and structural types libraries.
- **Response**: `200 OK` with JSON payload containing `architectural_types` and `structural_types`.

### 4. `POST /api/analyze`
Triggers the structural analysis pipeline (`run_e2e_flow.py`).
- **Response**: `200 OK` with `{"success": true, "message": "..."}`.
- **Error**: `500 Internal Server Error` if the pipeline fails or returns a non-zero exit code.

### 5. `GET /api/report`
Loads the structural report generated by the analysis pipeline.
- **Response**: `200 OK` with JSON payload containing the BOM and detailed element results.
- **Error**: `404 Not Found` if the report has not been generated yet.
